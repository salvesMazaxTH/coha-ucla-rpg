# Quick Task 260416-k87 - Summary

## Description

Corrigir a inconsistência visual de dano com escudo entre `intermediateSnapshot` e `animsAndLogManager`.

## Changes Implemented

- Backend (`05_applyDamage.js`):
  - Calcula `shieldBefore`, `remainingShield` e `absorbedByShield` no momento de aplicação de dano.
  - Passa a registrar `amount` como dano efetivo em HP (`event.actualDmg`) em vez do dano bruto.
  - Inclui metadados `rawAmount`, `absorbedByShield`, `remainingShield` no evento visual.

- Backend (`TurnResolver.js`):
  - `registerDamage` passou a aceitar/serializar os campos `rawAmount`, `absorbedByShield`, `remainingShield`.

- Frontend (`animsAndLogManager.js`):
  - Adicionada `parseVisualHpState` para extrair HP/HP max/escudo do HUD.
  - `animateDamage` agora usa dano líquido de HP para float e barra.
  - Quando há absorção por escudo, o HUD de escudo é atualizado imediatamente (sem esperar snapshot final).
  - `updateVisualHP` ganhou opções de controle de escudo (`shieldDelta`, `shieldOverride`).
  - `animateShield` já incrementa visualmente o escudo no HUD no momento da animação.

## Behavioral Result

- Exemplo esperado: alvo com 100 HP + 50 escudo recebendo 100 de dano:
  - Float mostra `-50` (ou indicador de bloqueio total quando dano em HP é 0).
  - Barra de HP vai direto para 50.
  - Escudo reduz imediatamente na UI.
  - Não ocorre mais efeito de HP descer para 0 e subir depois.

## Validation

- Diagnóstico do editor nos arquivos alterados: sem erros.
- Validação backend controlada (`applyDamage`) no cenário 100 HP + 50 escudo recebendo 100 de dano:
  - HP final: 50
  - Escudo final: 0
  - Evento emitido: `amount: 50`, `rawAmount: 100`, `absorbedByShield: 50`, `remainingShield: 0`
- Validação isolada da lógica visual (sem DOM real) confirmou transição do texto de HP:
  - `100/100 🛡️ (50)` -> `50/100`
