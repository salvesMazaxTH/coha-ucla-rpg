# Quick Task 260415-q1q: Lifesteal separado visualmente do heal comum

**Date:** 2026-04-15
**Status:** Completed

## Goal

Garantir que lifesteal continue passando pelo método `Champion.heal`, mas com identificação explícita no backend para registrar em `lifestealEvents` (via `registerLifesteal`) e animação dedicada no cliente (`animsAndLogManager`), sem conflitar com cura normal.

## Tasks

### 1. Tornar `heal` consciente do tipo de cura

- Files: `shared/core/championCombat.js`, `shared/core/Champion.js`, `shared/engine/combat/pipeline/07_afterHooks.js`
- Action: adicionar opção de tipo no `heal` (`normal`/`lifesteal`) e marcar o fluxo de lifesteal para usar essa opção
- Verify: lifesteal chama `Champion.heal(..., { type: "lifesteal" })` e `heal` roteia para `registerLifesteal` quando disponível
- Done: cura comum e lifesteal passam pelo mesmo método, porém com registro visual distinto

### 2. Propagar `lifestealEvents` no envelope do servidor

- Files: `src/server.js`
- Action: incluir `lifestealEvents` no cálculo de mudanças visuais e no payload enviado para o cliente
- Verify: `mainEnvelope` inclui `lifestealEvents` e dispara `emitCombatAction` quando somente lifesteal ocorre
- Done: cliente recebe eventos de lifesteal sem depender de `healEvents`

### 3. Implementar animação semântica de lifesteal no cliente

- Files: `public/js/animation/animsAndLogManager.js`, `public/styles/animations.css`
- Action: criar `animateLifesteal`, conectar no dispatcher e adicionar classes/efeitos CSS específicos
- Verify: `lifestealEvents` dispara animação dedicada (`.lifesteal`, `.lifesteal-float`) e não usa visual idêntico ao heal normal
- Done: feedback visual de lifesteal está distinto, intuitivo e coerente

## Outcome

- Lifesteal agora é identificado no backend antes do registro visual.
- Fluxo de HP permanece centralizado em `Champion.heal`.
- Frontend diferencia claramente cura normal de roubo de vida.
